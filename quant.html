1. 这是什么？

由香港大学数据智能实验室 (HKUDS) 开源的个人交易智能体（Personal Trading Agent），目标是把量化交易的整套流程（数据采集 → 策略研究 → 回测验证 → 实盘信号）通过 LLM Agent 串联起来，让"说人话就能跑量化"。

核心特点：多智能体自然语言研究α 动物园影子账户MCP 插件Docker 一键启动

与我们项目的并行关系：HKUDS Vibe-Trading 偏量化研究 / 回测 / 实盘，与我们已有的「盘前舆情 + 盘中 5 时段 + 盘后复盘 + 实时盯盘」四个功能模块平行，是第 5 个独立模块：量化分析。本项目只做单评估，暂不实盘接入。

📦 GitHub 仓库 🌐 官方文档

2. 核心功能（4 大块）

🤖

多智能体协作

Research / Strategy / Risk / Execution 多角色 Agent 协作，模拟真实投资团队的辩论 + 风控流程。

📝

自然语言研究

用自然语言描述策略思路（如"找 RSI 超卖 + 量能异动 + 行业景气"），Agent 自动拆解因子 + 拉数据 + 跑回测。

📊

α 动物园（Alpha Zoo）

预置 200+ 经典策略（动量 / 反转 / 因子 / 套利 / 统计套利），一行命令跑回测对比。

🛡️

影子账户（Shadow Account）

无风险实盘跟踪：信号不直接下单，先观察 N 天模拟表现，确认稳定后再切换实盘。

🔌

MCP 集成

作为 MCP 插件接入 WorkBuddy / Claude 等 LLM 客户端，可与我们专家的 71 个 skill 协同工作。

📡

多数据源智能降级

内置 Longbridge / QVeris 等数据源，免费源失败自动降级到付费源，确保回测无中断。

4. 快速安装（4 条路径）

路径 A · Docker（零配置 · 推荐）
git clone https://github.com/HKUDS/Vibe-Trading.git
cd Vibe-Trading
# 编辑 agent/.env — 取消注释你的 LLM 提供商并设置 API key
docker-compose up
路径 B · 本地安装（Python 3.11+）
python -m venv .venv
source .venv/bin/activate        # macOS/Linux
# .venv\Scripts\Activate.ps1   # Windows PowerShell
pip install -e .
# 终端 1: API server
# 终端 2: Frontend dev server
路径 C · MCP 插件（接入 WorkBuddy / Claude）
claude mcp add vibetrading -- pip install vibetrading-mcp
# 或在 .mcp.json 中添加：
# { "mcpServers": { "vibetrading": { "command": "python", "args": ["-m", "vibetrading.mcp"] } } }
路径 D · ClawHub（一键）
npx clawhub install HKUDS/Vibe-Trading

6. 投资研究工作流示例

Step 1 · 自然语言研究
vibe research "找 7 月份 A 股中 PE-TTM < 行业均值 + ROE > 15% + 北向连续 5 日净流入 + 股价突破 20 日均线的标的"
# Agent 自动拆解为：PE 因子 / ROE 因子 / 北向因子 / 动量因子 → 拉数据 → 跑回测
Step 2 · α 动物园回测
vibe bench --strategies momentum,factor,mean_reversion,stat_arb --start 2020-01-01
# 自动对比 200+ 经典策略，输出 Sharpe / 最大回撤 / 胜率 / IC / IR
Step 3 · 影子账户跟踪
vibe shadow --strategy my_alpha_v1 --capital 100000 --start 2026-08-01
# 无风险实盘跟踪 30 天，确认年化 Sharpe > 1.5 后再切换实盘

7. 风险提示

⚠️ 量化回测 ≠ 实盘盈利：历史回测表现优秀的策略，实盘可能因滑点 / 流动性 / 政策变化失效。

⚠️ 门槛不低：原作者明确表示"这不可能告诉怎么用"，使用前需具备：Python 编程 / 量化基础 / 金融工程思维。

⚠️ 实盘前必读：先用影子账户跟踪 30 天以上，确认 Sharpe > 1.5 / 最大回撤 < 15% 再考虑实盘，且仅用可承受损失的资金。

8. 5 持仓测试 · 实操案例

用本项目持有的 5 个标的（含用户提供的 4 个）演示 Vibe-Trading 三种典型用法。每种用法对应不同投资目标，按需选择：

📌 测试 1 · 银行板块趋势跟踪（招商银行 sh600036）

当前数据：价 38.71 / 涨 +1.90% / PE-TTM 8.2x / 不良率 0.95% / ROE 15%
Vibe-Trading 命令：
vibe research "A 股银行板块龙头 招商银行 2026 年下半年趋势，结合高股息防御主线 + LPR 持平/下调预期，给出 30 天回测策略"
# 自动拆解：PE 分位 / ROE 趋势 / 不良率历史 / 北向资金 / 行业景气
预期输出：3 个候选策略（动量 / 估值修复 / 红利低波）+ Sharpe / 最大回撤 / 胜率对比
适合场景：稳定派息、防御性配置、退休/保守型资金

📌 测试 2 · 指数定投择时（创业板指 sz399006）

当前数据：价 3,566.73 / 跌 -3.23% / PE-TTM 43x / 处于近 1 年 30% 分位
Vibe-Trading 命令：
vibe bench --strategies dip_buy,ma_cross,factor_etf --asset sz399006 --start 2018-01-01
# 自动对比回调买入/均线交叉/因子定投在创业板指上的 7 年表现
预期输出：3 个策略历史回测 + 资金曲线 + 最大回撤 / 当前 PE 分位 / 定投建议
适合场景：ETF/指数定投、估值分位择时、长期持有

📌 测试 3 · 个股事件驱动（紫光国微 sz002049）

当前数据：价 66.24 / 涨 +1.47% / 22nm 车规级 MCU 获车企定点 / 2026H1 车规业务 +230%

vibe event "紫光国微 22nm 车规 MCU 定点 5 年 8-12 亿营收 + 6 家车企累计定点，事件驱动量化分析"
